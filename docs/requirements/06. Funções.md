## Ferramentas e Gestão

O projeto será gerenciado utilizando práticas ágeis adaptadas para o contexto acadêmico.

- Controle de Versão e Código: Git e GitHub (Repositório `SUSi`).
- Gestão de Tarefas (Quadro Ágil): GitHub Projects (Kanban com: Backlog, To-Do, Doing, Done).
- Documentação: Markdown (integrado ao repositório).
- Comunicação: Reuniões diárias (Daily Scrum adaptado) de 15 minutos via plataforma de comunicação assíncrona.

## Estratégia de Desenvolvimento

O desenvolvimento será Data-Driven, priorizando a qualidade da base de dados antes da construção da interface. A estratégia divide-se em três fases técnicas:

1. Refinaria de Dados (Sprints 2 e 3): Foco total na construção do motor de ETL. O projeto depende de uma base de dados limpa e bem estruturada.
2. Inteligência Analítica (Sprint 4): Aplicação de métodos estatísticos e treinamento do modelo de machine learning para extrair valor dos dados processados.
3. Consumo e Visualização (Sprints 5 e 6): Construção da camada de apresentação (Dashboard) para que usuários não-técnicos possam interagir com os insights gerados.
