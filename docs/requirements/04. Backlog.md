# Backlog

## Épicos e Histórias de Usuário

### Épico 1: Estruturação do Pipeline de Dados (ETL)
É necessário extrair, limpar e estruturar dados fragmentados do SUS para criar uma base de análise confiável.

- US01: Criação de um script Python para extrair dados brutos do DATASUS em formato CSV, para iniciar o processamento.
- US02: Implementação de funções de limpeza no Pandas para tratar valores nulos e inconsistentes, garantindo a qualidade da base.
- US03: Cruzamento dos dados de agendamentos com a base do CNES (estabelecimentos), para adicionar informações geográficas aos registros.
- US04: Modelagem e instância de um banco de dados PostgreSQL, para armazenar a base limpa de forma relacional.

### Épico 2: Análise Exploratória e Modelagem Preditiva
É necessário analisar os dados limpos para provar estatisticamente as falhas estruturais e prever desfechos negativos.

- US05: Gerar estatísticas descritivas sobre o tempo de espera e distâncias percorridas, para fundamentar a análise do problema.
- US06: Treino de um modelo de Random Forest utilizando Scikit-Learn, para prever a probabilidade de desfechos negativos com base em variáveis estruturais.
- US07: Avaliação da acurácia e as métricas do modelo preditivo, para garantir que os resultados sejam estatisticamente válidos.

### Épico 3: Visualização e Dashboard (O Protótipo)
É necessário visualizar os dados processados em uma interface amigável para tomar decisões baseadas em evidências.

- US08: Criação da estrutura base de um dashboard utilizando Streamlit, para servir como interface do usuário.
- US09: Integração do banco de dados PostgreSQL ao dashboard, para que os gráficos consumam dados reais processados.
- US10: Implementação de gráficos interativos (Plotly) que mostrem gargalos por região e especialidade, para facilitar a interpretação visual pelo gestor.
- US11: Integração dos resultados do modelo preditivo no dashboard, para que o gestor possa visualizar áreas de alto risco de ineficiência.

## Priorização (Foco em Dados)
A prioridade inicial (WIP) será a execução do Épico 1, focando na consolidação da base de dados pois as etapas de modelagem e visualização dependem da qualidade dos dados processados (Refinaria de Dados).
